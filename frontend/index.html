① Matériel recommandé
Composants pour construire un collier GPS/GSM
📡
Module GPS
NEO-6M, NEO-M8N ou équivalent UART — précision 2–5 m
📱
Module GSM/GPRS
SIM800L, SIM7600 (4G) ou ESP32 avec SIM (LilyGO T-SIM7600) — carte SIM MTN/Orange CM
🔧
Microcontrôleur
ESP32 (recommandé) ou Arduino Mega + shield GSM séparé
🔋
Batterie + panneau solaire
LiPo 3 000–5 000 mAh + panneau 5 V/1 W — autonomie ≥ 7 jours
② Clé API & endpoint
Les deux seules informations à programmer dans le collier
URL de l'endpoint GPS
Le collier envoie une requête HTTP POST à cette adresse
POST /iot/gps
Clé API IoT
À inclure dans chaque requête du collier. Changer dans le fichier .env côté serveur.
geoalerte_iot_key_2024
⚠ Changer cette clé avant le déploiement en production — variable API_KEY_IOT dans .env
③ Format de la requête JSON
Corps exact à envoyer en POST
{
  "api_key":        "geoalerte_iot_key_2024",
  "id_troupeau":    1,
  "longitude":      13.602145,
  "latitude":       7.418320,
  "precision_pos":  4.5,
  "direction":      135,
  "niveau_batterie": 87
}

④ Code ESP32 (Arduino IDE)
Exemple complet à adapter — bibliothèques : TinyGPS++, HTTPClient
// ── Bibliothèques ──────────────────
#include <WiFi.h>
#include <HTTPClient.h>
#include <TinyGPS++.h>

// ── Configuration ──────────────────
const char* WIFI_SSID  = "VOTRE_WIFI";
const char* WIFI_PASS  = "MOT_DE_PASSE";
const char* SERVER_URL = "https://VOTRE_BACKEND/api/iot/gps";
const char* API_KEY    = "geoalerte_iot_key_2024";
const int   ID_TROUPEAU = 1;
const long  INTERVAL_MS = 60000; // 1 envoi / minute

// ── GPS : Serial2 (RX=16, TX=17) ──
TinyGPSPlus gps;
HardwareSerial gpsSerial(2);

void setup() {
  Serial.begin(115200);
  gpsSerial.begin(9600, SERIAL_8N1, 16, 17);
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500); Serial.print(".");
  }
  Serial.println("\nWiFi connecté");
}

unsigned long lastSend = 0;

void loop() {
  // Lire le GPS
  while (gpsSerial.available())
    gps.encode(gpsSerial.read());

  if (millis() - lastSend >= INTERVAL_MS
      && gps.location.isValid()) {
    sendPosition(
      gps.location.lng(),
      gps.location.lat(),
      gps.hdop.value() / 100.0,
      gps.course.deg(),
      readBatteryPercent()
    );
    lastSend = millis();
  }
}

void sendPosition(double lon, double lat,
                   float hdop, double course,
                   int batterie) {
  HTTPClient http;
  http.begin(SERVER_URL);
  http.addHeader("Content-Type", "application/json");
  String body = "{\"api_key\":\"" + String(API_KEY)
    + "\",\"id_troupeau\":" + ID_TROUPEAU
    + ",\"longitude\":"    + String(lon, 6)
    + ",\"latitude\":"     + String(lat, 6)
    + ",\"precision_pos\":"+ String(hdop, 1)
    + ",\"direction\":"    + String(course, 0)
    + ",\"niveau_batterie\":"+ batterie + "}";
  int code = http.POST(body);
  http.end();
  Serial.println(code == 200 ? "✓ Envoyé"
                             : "✗ Err " + String(code));
}

int readBatteryPercent() {
  // ADC sur GPIO34 (diviseur résistif 100k/47k)
  int raw = analogRead(34);
  float v = raw * 3.3f / 4095.0f * 3.14f;
  return constrain((v - 3.0f) / (4.2f - 3.0f) * 100, 0, 100);
}

Conseils mise en production
🔑 Sécurité de la clé API
Changer API_KEY_IOT dans le fichier .env du serveur. La clé est en clair dans le firmware — ne jamais la partager. En cas de compromission, changer uniquement côté serveur.
📶 Fréquence d'envoi recommandée
Transhumance active : 1 envoi / 5 min. Pâturage fixe : 1 envoi / 30 min. Nuit / repos : 1 envoi / heure. Adapter selon la consommation batterie.
📡 Couverture réseau au Cameroun
MTN CM et Orange CM couvrent les axes principaux de l'Adamaoua. En zone blanche, stocker les positions en mémoire flash et les transmettre par lot dès retour réseau.
✓ Vérifier l'intégration
Après le premier envoi réel : aller sur la page Carte en direct → le marqueur GPS du troupeau doit apparaître. Si le troupeau approche d'une zone, une alerte s'affiche automatiquement.
